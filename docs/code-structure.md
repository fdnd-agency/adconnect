## Code structure

We follow the FDND code conventions to keep the code consistent and easy to read.  
The full rules are described here:

- [FDND conventions](https://docs.fdnd.nl/conventies.html)
- [Contributing file (FDND conventions section)](https://github.com/fdnd-agency/adconnect/blob/dev/CONTRIBUTING.md#fdnd-conventions)

Important points we follow:

- Use meaningful names and write everything in English.
- Use kebab-case for CSS class names and ids.
- Use camelCase for JavaScript variables and functions.
- Use single quotes in JavaScript.
- Use `const` by default, `let` only when needed.
- Use semantic HTML and avoid deep nesting.
- Fetch data in `+page.server.js` and keep pages clean by using components.
- Use conventional commits and Git Flow for the workflow.

These rules help keep the project consistent and easy for everyone to work on.
